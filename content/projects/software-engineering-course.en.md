## Overview

This course presentation explains the iterative software process model. It starts from the need for software process models, then introduces the core idea, workflow, strengths, limitations, and suitable scenarios of iterative development.

## Structure

- Software process model overview.
- Problems in traditional development, including changing requirements, concentrated risks, and late user feedback.
- Definition of the iterative model.
- Iteration workflow: requirement analysis, design, implementation, testing, and release.
- Case analysis based on continuously evolving software products.
- Reflection on advantages, limitations, and suitable use cases.
